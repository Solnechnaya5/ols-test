What happens when you open a website or an app?

You see a page in a browser or smartphone scream. It's content, visual design, animations, various components of a page that was made to interact with you as a user and show all necessary information.

All of this and many more are developing using frontend development. Today we would like to show how we create frontend development in our company and who is frontend developer.

We offer to start from the very beginning, so let's look closer at what is Frontend?

Frontend is everything that a user can see and interact with on Web (in Web Browser), in Desktop programs etc. Basically, Frontend provides a user interface which is made with special markup language HTML, styles using CSS, and programming language. Any browser page dynamic reactions on user's click or even mouse movement are implemented by Javascript or other programming language using special approaches and frameworks.

What is important when you create a web page or application ( according to the frontend development)

One of the important things is to be attentive to task description and at the same time do it according to your estimation. Second principal thing is communication with the team. If you communicate well with management and other devs, that's quite a growth in the success percentage, so don't be afraid to tell your team about your issues or even successes. Third main point is to keep code simple and clear for other devs in your team or the next team. We have a lot of principles to make code easy to understand.

What most common programming languages are using for frontend development, their advantages and disadvantages?

Approximately 80% of the Web is written using the PHP programming language. But, of course, any UI (User-Interface) includes HTML and CSS. As for PHP for now, I guess that percentage will grow down, cause now many developers think that PHP is a bit legacy, which means it's too old, however it's updating and supporting until nowadays.

But most new web developers learn JavaScript as their first programming language so perhaps JavaScript will gain some speed in that race. As for 2023 I guess that JavaScript is still the main frontend language, despite the idea to make a substitution for it. Also JavaScript frameworks and libraries as React, Angular and Vue gain momentum pretty fast.

What frameworks and libraries are you using in your day to day work?

Every day I use HTML(jsx or tsx - in React HTML, to say it clearly for all, is mixed with Javascript/Typescript), CSS(mostly with preprocessor SASS(SCSS)), Javascript/Typescript, React, Redux (for maintaining and updating data and control the data flow in frontend web app), Redux Thunk, Git (to control versions of application for example: include some bug fixes or new features to next version of your application).

But also I have experience with some more technologies such as jQuery, PHP, Wordpress, Laravel, Web Components, Vue js and Angular js. When I was developing extensions I worked with Browser API a lot.

How do you make the page look good both in browsers and on mobile devices?

In CSS we have special “commands” to set the styles for mobile, tablet, small laptop screens. So on mobile browsers or PWA built with the help of Cordova or some other library we use commands named Media Queries.

What is React.js and why is it good for frontend development?

React.js is a library which helps us to improve the updating of data in our SPA(single page application). React provides high performance - core of React js library offers us virtual DOM which React is famous for.

Easy to learn, in this way React helps businesses to faster build the projects because JavaScript developers can easily learn how to use React.

SEO-friendly - because of fast load time of the page, your website will appear on higher rank on Google.
